Semantic Markup

April 1, 2021 by Mandaha RS


The word semantic has to do with language and logic while Markup is the process of text correction furtherly, semantic markup
is simply the correct interpretation of the meaning of words or sentence. It also referred as a way of writing and structuring
HTML so that it supports the meaning of the content, therefore for your site to make sense it should be well structured.
Using semantic markup shows that you know and understand the system or layers of the content you are sharing to users as well as the machines.

We have HTML already stated above which is a semantic markup language ,when HTML tags are used they focus on the purpose
rather than the appearance that is what we mean semantic markup for an example, the tag(a)is used to create links while the arrows
on both sides show break in content. When creating our websites, the objective is to describe the content being shared
clearly using tags accessible. The tags make it easier for developers to differentiate types of data and create content in a way that
computers will be able to read through, some tags include (header, footer, p (paragraph tag)) etc.

Why Semantic Markup is important


When creating content it is important to think of the users who will go through what have shared and the computers that will read your content,
how your work is structured, it must easy to navigate around your site and understand the information on your site so yes to achieve this,
you need to use semantic markup. When creating you must have a target group so that in that way you will know the value of
your website because you will ensure that it reaches the target group, therefore knowing the importance of keyword
placement according to their value will guide the creator on ensuring that websites are easily accessible.

Communication is vital when it comes to creating content therefore, using semantic code produces a code that is easy to navigate and it is effective
in terms searching through it and it will be easy to read like which line does what? and can be used by the next person maybe to
develop it further or to learn something from it. Following semantic markup helps to build meaningful websites, your work must be
used in the future and for that your code must be well structured and validated. Search engines should be able to figure out what our documents
are about to have access to your content pages. A well structured content can be easily updated, the style can also be changed without data interruption.

The relation between HTML, CSS and JavaScript (Website creation)

may 20, 2021 by Mandaha RS


To provide and understanding of the topic, I will start by giving a description of each language mentioned above.
Firstly, we must understand that HTML and CSS are not programming languages, HTML is a Hypertext Markup Language
which is used in structuring the web and layering content so that it has logic, using tags that
perform certain functions on the design of webpages. CSS stands for Cascading Style Sheets and has
to with styling, it is used to change colours of elements, width and height, crating animation,
transformation, decoration of elements, creating columns, images, sizes of elements
and how you would want them to be presented on a website. JavaScript is a programming language which
is understood by the browser, it helps in making web pages interactive, it has to do with usability
and an example would be a search box on a website so basically
clicking on an object for it to respond (responsiveness).
All these languages are used in designing and development of websites, they work together to bring functionality on the web. JS is able to make
changes on both HTML and CSS, it manipulates them by changing the behaviour, display and interaction of the web page.
A JS script is linked to HTML script and in that way, information is made available and changed can be made on HTML code,
while using JS and CSS make images to load faster by using CSS rule once on a tag for using less code makes
images to load fast, style sheets allow content effective on different devices. JS is used on the page
for interaction and some examples are zooming in and zooming out images, showing animations, playing audio or videos,
clicking on a button for it to do something, etc. all this can also be done using HTML
and CSS but adding JS just make usability more effective and the web
page to become more operative.
How do all of them work together to create a functional website? The browser will start by reading HTML and create a content tree thereafter,
it looks and read through CSS and apply it to the content tree to create a render tree which old styling
in it, then it goes through HTML again which have all the elements then after we are able everything displayed.
JS is analysed by the browser then it make changes on the existing Dom components and manipulate
how the CSS relates to them. The browser has CSS and HTML parser so this will read the HTML text and CSS style tags,
then selectors of CSS will be combined with HTML Dom tree to create a render tree. The render
tree renders the website and compute a geometry for each node. Think of using these languages as a
way of offloading weights on one script to another and bring balancing on the website, know when
to use CSS, same goes with HTML and CSS.

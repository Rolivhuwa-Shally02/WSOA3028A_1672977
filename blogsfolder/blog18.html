The use of Web APIs

June 30, 2021 by Mandaha RS



To understand what Web API is, let’s first look into the definition of API itself, API stands for Application Programming Interface, it is the code that governs the access
points for the server, it provides data so that you can view or use an application. Proper and good APIs make it easier to
develop a computer program by providing all the building blocks, which are then put together by the programmer. Another
definition which can make things understandable is that Application Programming Interfaces are
constructs made available in programming languages to allow developers to create complex functionality more
easily. What they do is that they abstract more complex code away from you, giving back easier syntax that you
can understand and make use of.
Here is an example from MDN Web Docs-electricity supply in your house, apartment, or other dwellings. If you want to use an appliance, you will plug it into a plug socket and it does
work right? you would not have to try and wire it directly into the power supply — to that would not be efficient and, if you
are not an electrician or do not know your ways around operation of electric appliances, it is difficult and dangerous to attempt. Now from that example you will be able to analyse
and understand why we use Web API and why they are necessary for your Websites and Applications. I mean if you can get google APIs to your site
without having to create yours is very convenient, yes you have add more codes depending on how you want they API to look on your website.
As we move deeper to why Web APIs are necessary, why we use them and different approaches that I know, we will also get know what APIs can do basically, their use and
importance. The browser APIs can be used to manipulate documents, the DOM API allows the manipulation of HTML and CSS, you can create, remove
and change HTML, apply your styling to get the layout you prefer. APIs are used to fetch data from server like what I used, while also
following a lecture form class. We used fetch to get data from a server. Where I used Rapidapi.com site to check for APIs that were accessible/free and then use my provided
unique API key. Fetch is used on small detail parts of the page, you can use the Fetch API to get more functions of the same API
and even add few more APIs to your website. You can integrate this with JavaScript, HTML and your CSS. Web APIs are also important when
you want to add video APIs, playing Audio, displaying tracks and you can also manipulate this to operate the way you would prefer. They can also be useful
when you want to store information on the client side, like making the app to show some data even when you are offline.
I also the importance of Web APIs when I had planned to create my own food search Application, while with Web API you can look in the internet for already made APIs, either free,
freemium, or prepaid so you can select what you prefer and fetch that to your website and manipulate it, add codes, style the layout. It is efficient
and effective especially if that API is accessible at all times for the users. I used COVID-19 statistics API from Rapidapi.com, I chose this API
because it is relevant to health and what is happening world wide as we are still in pandemic with a rise in Covid cases, I believe that people will like to know about that
information but most importantly it adds to my website content, I wanted more interaction in my site and on the about
page, reason I put the Web API there. Users are able to check the statistics of every country, I fetched data for new confirmed cases, total
confirmed cases, new deaths, total deaths and last updated (which basically shows the current date).
